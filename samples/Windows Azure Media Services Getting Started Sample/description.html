Introduction

The code examples in this project show a first-time developer on Windows Azure Media Services how to complete a simple end-to-end workflow of creating an asset, uploading it, encoding it, and then downloading the output asset.  The code examples are written in C#, and the project uses the Media Services SDK for .NET.  The project is a Windows Console application and is designed to accompany the following Getting Started tutorial on MSDN. The documentation also provides information on the setup and connection process: Getting Started with the Media Services SDK for .NET topic.

Building the sample

After you download and extract the files for the sample, there are several steps required to set up and run the project.

 For more information, see Getting Started with the Media Services SDK for .NET topic. 

  • A Media Services account in a new or existing Windows Azure subscription. See the topic How to Create a Media Services Account. 
  • Operating Systems: Windows 7, Windows 2008 R2, or Windows 8.
  • .NET Framework 4.5 or .NET Framework 4.
  • Visual Studio 2012 or Visual Studio 2010 SP1 (Professional, Premium, or Ultimate).
  • Use the windowsazure.mediaservices Nuget package to install and add references to: Windows Azure SDK for .NET, Windows Azure Media Services SDK for .NET (remove any previous versions of the Media Services SDK before installing the current release ), and WCF Data Services 5.0 for OData V3.
  • Add an appSettings section to the app.config file, and set the values for your Media Services account name and account key. You obtained the Media Services account name and account key during the account setup process. Add these values to the value attribute for each setting in the app.config file in the Visual Studio project.
